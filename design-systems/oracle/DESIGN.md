# Design System Inspired by Oracle

> Category: Enterprise & Cloud Computing
> Bold, enterprise-grade design with Oracle's signature red, professional typography, and cloud-focused aesthetic for database and enterprise software solutions.

## 1. Visual Theme & Atmosphere

Bold, enterprise-grade design with Oracle's signature red, professional typography, and cloud-focused aesthetic for database and enterprise software solutions.

- **Visual style:** enterprise, professional, cloud-focused, database-centric
- **Color stance:** signature red primary, neutral grays, enterprise blue accents
- **Design intent:** Convey reliability, enterprise scale, and technical excellence while maintaining approachability.

## 2. Color

- **Primary:** `#C74634` — Oracle signature red for CTAs and brand emphasis.
- **Secondary:** `#312D2A` — Deep charcoal for headers and strong contrast.
- **Success:** `#1B7F37` — Enterprise green for positive states.
- **Warning:** `#F5A623` — Amber for warnings and alerts.
- **Danger:** `#D0312D` — Red variant for errors and critical states.
- **Surface:** `#FFFFFF` — Clean white for backgrounds and cards.
- **Text:** `#312D2A` — Dark charcoal for body text.
- **Neutral:** `#F5F5F5` — Light gray for subtle backgrounds.
- **Accent:** `#0572CE` — Oracle blue for cloud and technical elements.

- Favor Primary (#C74634) for CTA emphasis and brand moments.
- Use Accent (#0572CE) for cloud services and technical features.
- Keep body copy on Text (#312D2A) for maximum legibility.
- Use Neutral (#F5F5F5) for section backgrounds and subtle separation.

## 3. Typography

- **Scale:** enterprise scale with clear hierarchy
- **Families:** primary=Inter, display=Inter, mono=Roboto Mono
- **Weights:** 400, 500, 600, 700, 800, 900
- **Sizes:** 14/16/18/20/24/32/40/48/56px
- Headings should be bold and confident; body text should be highly readable.
- Use font-weight 700-900 for headlines to convey enterprise strength.

## 4. Spacing & Grid

- **Spacing scale:** 8pt baseline grid (8, 16, 24, 32, 40, 48, 64, 80, 96, 128px)
- **Grid:** 12-column responsive grid with consistent gutters
- Keep vertical rhythm consistent across sections and components.
- Use generous spacing to create breathing room and enterprise feel.

## 5. Layout & Composition

- Prefer structured layouts with clear content blocks and consistent padding.
- Keep hierarchy obvious: headline → subheadline → supporting text → primary action.
- Use whitespace generously to separate major sections.
- Align to grid for predictable, professional layouts.

## 6. Components

- **Buttons:** primary uses #C74634 with white text; secondary uses white with red border; tertiary uses #0572CE for cloud actions.
- **Cards:** clean white backgrounds with subtle shadows, 8px border-radius, consistent padding.
- **Inputs:** strong borders, clear focus states, enterprise-grade validation.
- **Navigation:** clean, horizontal, with clear active states.

## 7. Motion & Interaction

- Use subtle, professional transitions (200-300ms) with ease-in-out.
- Hover states should be clear but not distracting.
- Focus-visible states must be strong for accessibility.
- Button interactions should feel solid and responsive.

## 8. Voice & Brand

- Tone: confident, technical, enterprise-focused, solution-oriented.
- Emphasize reliability, scale, performance, and enterprise capabilities.
- Use technical language appropriately but remain accessible.
- Highlight cloud infrastructure, database power, and enterprise solutions.

## 9. Anti-patterns

- Do not use off-brand colors that dilute Oracle's signature red identity.
- Do not create cluttered layouts that undermine enterprise professionalism.
- Do not use playful or casual design elements inappropriate for enterprise software.
- Do not sacrifice readability for visual effects.
