# Databricks Design System

## Overview
Databricks design system emphasizes data lakehouse architecture, unified analytics, and AI/ML capabilities with a technical, enterprise-focused aesthetic.

## Brand Identity
- **Personality**: Technical, innovative, data-driven, enterprise-grade
- **Voice**: Professional, authoritative, forward-thinking
- **Focus**: Unified data analytics, lakehouse architecture, AI/ML integration

## Color Palette

### Primary Colors
- **Databricks Red**: `#FF3621` - Primary brand color, CTAs, highlights
- **Databricks Orange**: `#FF6F3C` - Secondary accent, gradients
- **Deep Navy**: `#1B3139` - Headers, text, professional depth

### Secondary Colors
- **Lake Blue**: `#00A4BD` - Data lake theme, accents
- **Teal**: `#00C9A7` - Success states, data visualization
- **Purple**: `#7B61FF` - AI/ML features, innovation
- **Coral**: `#FF6B6B` - Warnings, alerts

### Neutral Colors
- **White**: `#FFFFFF` - Backgrounds, cards
- **Light Gray**: `#F7F9FA` - Secondary backgrounds
- **Medium Gray**: `#E5E9EB` - Borders, dividers
- **Dark Gray**: `#4A5568` - Secondary text
- **Charcoal**: `#2D3748` - Body text

## Typography

### Font Families
- **Primary**: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **Display**: `'Inter', sans-serif` (weight 700-900)
- **Monospace**: `'Fira Code', 'Monaco', 'Courier New', monospace`

### Type Scale
- **Display**: 56px / 700-900 weight
- **H1**: 48px / 700 weight
- **H2**: 40px / 700 weight
- **H3**: 32px / 600 weight
- **H4**: 24px / 600 weight
- **Body Large**: 18px / 400 weight
- **Body**: 16px / 400 weight
- **Small**: 14px / 400 weight
- **Caption**: 12px / 400 weight

### Line Heights
- **Headings**: 1.2
- **Body**: 1.6
- **Code**: 1.5

## Spacing System
8px base unit grid system:
- **xs**: 4px
- **sm**: 8px
- **md**: 16px
- **lg**: 24px
- **xl**: 32px
- **2xl**: 48px
- **3xl**: 64px
- **4xl**: 96px
- **5xl**: 128px

## Components

### Buttons
- **Primary**: Red background (#FF3621), white text, 8px border-radius
- **Secondary**: White background, red border, red text
- **Tertiary**: Transparent, red text, underline on hover
- **Sizes**: Small (32px), Medium (40px), Large (48px)
- **States**: Hover (darken 10%), Active (darken 20%), Disabled (50% opacity)

### Cards
- **Background**: White
- **Border**: 1px solid #E5E9EB
- **Border Radius**: 12px
- **Shadow**: 0 2px 8px rgba(0,0,0,0.08)
- **Hover**: Lift with shadow 0 4px 16px rgba(0,0,0,0.12)
- **Padding**: 24px

### Navigation
- **Height**: 64px
- **Background**: White with subtle shadow
- **Logo**: Databricks red
- **Links**: Dark gray, red on hover
- **Sticky**: Yes

### Forms
- **Input Height**: 40px
- **Border**: 1px solid #E5E9EB
- **Border Radius**: 6px
- **Focus**: 2px red outline
- **Label**: 14px, medium gray, 8px margin-bottom

### Data Visualization
- **Chart Colors**: Red, Orange, Lake Blue, Teal, Purple
- **Grid Lines**: Light gray (#E5E9EB)
- **Tooltips**: Dark background, white text
- **Animations**: Smooth transitions, 300ms ease

## Layout

### Grid System
- **Columns**: 12-column grid
- **Gutter**: 24px
- **Max Width**: 1280px
- **Breakpoints**:
  - Mobile: 0-767px
  - Tablet: 768-1023px
  - Desktop: 1024-1439px
  - Wide: 1440px+

### Sections
- **Padding**: 80px vertical, 24px horizontal
- **Background**: Alternating white and light gray
- **Max Width**: 1280px centered

## Iconography
- **Style**: Outlined, 2px stroke
- **Sizes**: 16px, 20px, 24px, 32px
- **Color**: Inherit from parent or red for emphasis
- **Set**: Custom Databricks icons + Feather Icons

## Imagery
- **Style**: Technical, data-focused, abstract visualizations
- **Overlays**: Dark gradient overlays for text readability
- **Aspect Ratios**: 16:9 for hero, 4:3 for cards
- **Treatment**: Subtle blur or gradient overlays

## Animation & Transitions
- **Duration**: 200ms (fast), 300ms (standard), 500ms (slow)
- **Easing**: ease-in-out for most, ease-out for entrances
- **Hover**: Scale 1.02, lift 4px
- **Page Transitions**: Fade in 300ms
- **Loading**: Spinner with red accent

## Accessibility
- **Contrast Ratio**: Minimum 4.5:1 for text
- **Focus States**: 2px red outline, 2px offset
- **Keyboard Navigation**: Full support
- **Screen Readers**: ARIA labels on all interactive elements
- **Motion**: Respect prefers-reduced-motion

## Voice & Tone
- **Technical**: Use precise, data-oriented language
- **Confident**: Assert expertise in data and AI
- **Clear**: Avoid jargon, explain complex concepts simply
- **Forward-thinking**: Emphasize innovation and future capabilities

## Key Messaging
- Unified data lakehouse platform
- Open architecture and interoperability
- AI and machine learning at scale
- Enterprise-grade security and governance
- Collaborative data science and engineering

## Design Principles
1. **Data-First**: Every design decision supports data workflows
2. **Unified Experience**: Seamless integration across analytics, engineering, and ML
3. **Enterprise-Grade**: Professional, reliable, scalable
4. **Open & Collaborative**: Support for open standards and team collaboration
5. **Innovation-Driven**: Showcase cutting-edge AI/ML capabilities
